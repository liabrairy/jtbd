# Настройка Яндекс Метрики для сайта книги

Документ описывает, что нужно сделать в интерфейсе Яндекс Метрики после подключения счетчика в коде проекта.

## Что уже сделано в проекте

- В код сайта подключен счетчик Яндекс Метрики `109588087`.
- Счетчик подключается в Next.js-сборке и в автономном HTML-файле.
- В коде уже отправляются полезные действия через `ym(..., "reachGoal", ...)`.
- Для переключения глав дополнительно отправляется виртуальный просмотр страницы с URL вида `/?chapter=01_chapter.md`.

Код счетчика вручную в интерфейсе копировать и вставлять больше не нужно.

## 1. Проверить настройки счетчика

1. Откройте Яндекс Метрику.
2. Выберите счетчик `109588087`.
3. Перейдите в `Настройка`.
4. На вкладке `Счетчик` проверьте адрес сайта.

Для GitHub Pages ожидаемый адрес:

```text
https://kartbllansh.github.io/TranslateJTBDBook/
```

Если сайт будет опубликован на другом домене, добавьте этот домен в адреса счетчика.

Проверьте, что включены настройки, которые используются в коде:

- Вебвизор.
- Карта кликов.
- Отслеживание внешних ссылок.
- Точный показатель отказов.

Параметр `ecommerce: "dataLayer"` также передается в коде счетчика, но сейчас сайт книги не отправляет ecommerce-события. Специально настраивать ecommerce для текущей читалки не обязательно.

## 2. Создать цели

Перейдите в `Цели` в левом меню счетчика и создайте цели типа `JavaScript-событие`.

Для каждой цели:

1. Нажмите `Добавить цель`.
2. В поле `Название` укажите понятное название из таблицы ниже.
3. Выберите тип `JavaScript-событие`.
4. В поле идентификатора укажите значение из колонки `ID события`.
5. Если интерфейс показывает условие, оставьте вариант точного совпадения идентификатора.
6. Нажмите `Добавить цель`.

| Название цели | ID события | Когда срабатывает |
| --- | --- | --- |
| Поиск по книге | `search_used` | Пользователь начал вводить поисковый запрос. |
| Дочитал 50% главы | `read_50` | Пользователь дошел до 50% текущей главы. |
| Дочитал главу | `read_100` | Пользователь дошел до конца текущей главы. |
| Клик по Telegram | `telegram_click` | Пользователь нажал на ссылку Telegram переводчика. |

## 3. Что не нужно создавать как цель

Не создавайте цель `chapter_open`. Открытие главы само по себе не является полезной конверсией для этого проекта.

Открытия глав остаются в аналитике как виртуальные просмотры страниц. Это лучше подходит для отчета по чтению: можно увидеть, какие главы открывают чаще, без лишней цели в списке конверсий.

При открытии главы сайт отправляет `hit` с URL:

```text
/?chapter=01_chapter.md
```

Не создавайте отдельные JavaScript-цели `chapter_open_01`, `chapter_open_02` и так далее. Текущий код их не отправляет.

## 4. Как смотреть открытия глав

Для разбивки по конкретным главам используйте отчеты по страницам. Фильтруйте URL по `chapter=`.

Примеры:

| Что найти | Фильтр URL |
| --- | --- |
| Все открытия глав | `chapter=` |
| Глава 1 | `chapter=01_chapter.md` |
| Глава 2 | `chapter=02_chapter.md` |
| Приложение 15 | `chapter=15_appendix.md` |

Если когда-нибудь понадобится отдельная конверсия на конкретную главу, используйте цель типа `Посещение страницы`, а не JavaScript-событие.

Пример настройки:

1. Нажмите `Добавить цель`.
2. Укажите название, например `Открытие главы 1`.
3. Выберите тип `Посещение страницы`.
4. Выберите условие `url: содержит`.
5. Укажите значение, например:

```text
chapter=01_chapter.md
```

6. Нажмите `Добавить цель`.

## 5. Проверить работу счетчика

После деплоя откройте опубликованный сайт.

Для проверки через отладочную панель добавьте к адресу параметр:

```text
?_ym_debug=2
```

Пример:

```text
https://kartbllansh.github.io/TranslateJTBDBook/?_ym_debug=2
```

Проверьте:

1. На странице появилась отладочная панель Метрики.
2. Во вкладке `Counters` виден счетчик `109588087`.
3. При переходе между главами появляются события `hit`.
4. При поиске появляется событие `search_used`.
5. При клике по Telegram появляется событие `telegram_click`.
6. При переходе между главами в URL события `hit` появляется параметр `chapter=...`.

Если отладочная панель не появилась, проверьте страницу в режиме инкогнито и отключите блокировщики рекламы.

## 6. Проверить цели в отчетах

Через несколько минут после тестовых действий:

1. Откройте счетчик `109588087`.
2. Перейдите в раздел `Цели`.
3. Проверьте, что у созданных целей начали появляться достижения.
4. Откройте отчеты по страницам и найдите URL с параметром `chapter=`.

Для проверки открытия конкретной главы используйте фильтр URL, например:

```text
chapter=01_chapter.md
```

## 7. Если цели не срабатывают

Проверьте по порядку:

1. На сайте опубликована свежая сборка.
2. В интерфейсе Метрики выбрана цель типа `JavaScript-событие`.
3. В идентификаторе цели нет опечатки.
4. Условие цели - `совпадает`.
5. ID счетчика в интерфейсе и в коде один и тот же: `109588087`.
6. В браузере не включен AdBlock или другой блокировщик счетчиков.
7. В настройках счетчика нет фильтра `Не учитывать мои визиты`; если он есть, проверяйте в инкогнито.
