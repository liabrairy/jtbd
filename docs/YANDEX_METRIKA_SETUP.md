# Настройка Яндекс Метрики для сайта книги

Документ описывает, какие цели нужно создать в Яндекс Метрике и как смотреть прогресс чтения по главам.

## Что уже сделано в коде

- Подключен счетчик Яндекс Метрики `109588087`.
- Счетчик подключается в Next.js-сборке и в автономном HTML-файле.
- При открытии главы отправляется виртуальный просмотр страницы с URL вида `/?chapter=01_chapter.md`.
- При чтении основной главы отправляются цели `book_started`, `chapter_read`, `book_completed`.
- При поиске и клике по Telegram отправляются цели `search_used` и `telegram_click`.
- Прочитанные главы запоминаются в `localStorage`, но в Метрику отправляется не сам `localStorage`, а рассчитанные параметры: номер главы, количество прочитанных глав и процент книги.

Код счетчика вручную в интерфейсе копировать и вставлять больше не нужно.

## Основная идея

Главная аналитическая цель - `chapter_read`.

Она срабатывает один раз на каждую основную главу для конкретного браузера, когда читатель дошел до 95% главы. Основными считаются файлы вида `NN_chapter.md`: сейчас это главы 1-15. Предисловия, приложения и примечания не входят в счетчик прочитанных глав.

Когда читатель дочитал главу, сайт отправляет событие:

```js
ym(COUNTER_ID, "reachGoal", "chapter_read", {
  book_id: "when-coffee-and-kale-compete-ru",
  reader_id: "anonymous-browser-id",
  chapter_file: "03_chapter.md",
  chapter_number: 3,
  chapter_title: "Глава 3. ...",
  chapters_read_count: 3,
  total_main_chapters_count: 15,
  book_read_percent: 20,
  read_chapters: "01,02,03",
  is_book_completed: "no"
});
```

`reader_id` - анонимный ID браузера, а не персональные данные пользователя.

## 1. Проверить настройки счетчика

1. Откройте Яндекс Метрику.
2. Выберите счетчик `109588087`.
3. Перейдите в `Настройка`.
4. На вкладке `Счетчик` проверьте адрес сайта.

Для GitHub Pages ожидаемый адрес:

```text
https://kartbllansh.github.io/TranslateJTBDBook/
```

Проверьте, что включены настройки, которые используются в коде:

- Вебвизор.
- Карта кликов.
- Отслеживание внешних ссылок.
- Точный показатель отказов.

## 2. Создать цели

Перейдите в `Цели` в левом меню счетчика и создайте цели типа `JavaScript-событие`.

| Название цели | ID события | Когда срабатывает |
| --- | --- | --- |
| Начал читать книгу | `book_started` | Читатель дошел минимум до 10% любой основной главы. |
| Дочитал главу | `chapter_read` | Читатель дошел до 95% основной главы, и эта глава еще не засчитывалась этому браузеру. |
| Дочитал книгу | `book_completed` | Читатель дочитал все 15 основных глав. |
| Поиск по книге | `search_used` | Читатель начал вводить поисковый запрос. |
| Клик по Telegram | `telegram_click` | Читатель нажал на ссылку Telegram переводчика. |

Для каждой цели:

1. Нажмите `Добавить цель`.
2. Укажите название из таблицы.
3. Выберите тип `JavaScript-событие`.
4. В идентификаторе укажите ID события из таблицы.
5. Оставьте точное совпадение идентификатора.
6. Сохраните цель.

Старые цели `read_50` и `read_100` текущий код больше не отправляет. Их можно оставить для исторических данных или удалить/архивировать, если в интерфейсе они мешают.

## 3. Как смотреть дочитывание глав

Основной отчет:

```text
Отчеты -> Содержание -> Параметры событий
```

Важно: для появления параметров из `reachGoal` нужно создать отдельную JavaScript-цель `chapter_read`. Если цель не создана, параметры `chapter_number`, `chapters_read_count` и другие могут не появиться в отчете "Параметры событий".

В отчете выберите цель `chapter_read` и группируйте данные по параметрам.

### Воронка по главам

Группировка:

```text
chapter_number
```

Так видно, какие главы дочитывают и где аудитория отваливается.

Пример ожидаемого смысла отчета:

| `chapter_number` | Достижения `chapter_read` |
| ---: | ---: |
| 1 | 120 |
| 2 | 73 |
| 3 | 41 |
| 4 | 28 |

### Сколько глав читает человек

Группировка:

```text
chapters_read_count
```

Так видно накопленное количество прочитанных глав к моменту каждого события.

Пример:

| `chapters_read_count` | Что значит |
| ---: | --- |
| 1 | Читатель дочитал свою первую уникальную главу. |
| 2 | Читатель дочитал две уникальные главы. |
| 3 | Читатель дочитал три уникальные главы. |
| 15 | Читатель дочитал все основные главы. |

### Какие главы уже были прочитаны

Группировка:

```text
read_chapters
```

Примеры значений:

```text
01
01,02
01,02,03
01,02,03,04
```

Это удобно, если человек читает подряд. Если он читает не по порядку, значение покажет фактически засчитанные главы, например `01,05,08`.

### Отдельный анонимный читатель

Группировка или фильтр:

```text
reader_id
```

Это не пользовательский аккаунт и не персональные данные. Это случайный ID, сохраненный в браузере. Он поможет разбирать отдельные траектории чтения, но основной отчет лучше строить не по нему, а по `chapter_number` и `chapters_read_count`.

## 4. Какие параметры отправляются

| Параметр | Что означает |
| --- | --- |
| `book_id` | ID книги: `when-coffee-and-kale-compete-ru`. |
| `reader_id` | Анонимный ID браузера. |
| `chapter_file` | Файл дочитанной главы, например `03_chapter.md`. |
| `chapter_index` | Индекс файла в списке читалки. |
| `chapter_label` | Метка главы в интерфейсе, например `03`. |
| `chapter_number` | Номер основной главы: `1`-`15`. |
| `chapter_title` | Заголовок главы. |
| `chapter_kind` | Сейчас для `chapter_read` всегда `main`. |
| `progress` | Прогресс скролла, при котором засчиталось событие. Обычно `95` или выше. |
| `chapters_read_count` | Сколько уникальных основных глав уже засчитано этому браузеру. |
| `total_main_chapters_count` | Общее количество основных глав: `15`. |
| `book_read_percent` | Накопленный процент книги по основным главам. |
| `read_chapters` | Список засчитанных глав: `01,02,03`. |
| `is_book_completed` | `yes`, если дочитаны все основные главы, иначе `no`. |

## 5. Параметры посетителей

После `book_started` и `chapter_read` сайт также отправляет `userParams`:

```js
ym(COUNTER_ID, "userParams", {
  UserID: reader_id,
  book_id: "when-coffee-and-kale-compete-ru",
  chapters_read_count: 3,
  book_read_percent: 20,
  last_read_chapter: "03_chapter.md",
  read_chapters: "01,02,03",
  is_book_completed: "no"
});
```

Смотреть можно здесь:

```text
Отчеты -> Содержание -> Параметры посетителей
```

Данные `userParams` могут появляться не сразу, а через несколько часов. Для оперативной проверки используйте отчет `Параметры событий`.

## 6. Открытия глав

Открытия глав остаются виртуальными просмотрами страниц, а не отдельными целями. При открытии главы сайт отправляет `hit` с URL:

```text
/?chapter=01_chapter.md
```

Это можно смотреть в отчетах по страницам, фильтруя URL по `chapter=`.

Не создавайте отдельные цели `chapter_open_01`, `chapter_open_02` и так далее. Для дочитывания конкретной главы уже есть одна цель `chapter_read` и параметр `chapter_number`.

## 7. Проверка после деплоя

Откройте опубликованный сайт с отладочным параметром:

```text
https://kartbllansh.github.io/TranslateJTBDBook/?_ym_debug=2
```

Проверьте:

1. В отладочной панели Метрики виден счетчик `109588087`.
2. При открытии глав появляются события `hit` с `chapter=...`.
3. При вводе поиска появляется `search_used`.
4. При клике по Telegram появляется `telegram_click`.
5. При чтении основной главы до 10% появляется `book_started`.
6. При чтении основной главы до 95% появляется `chapter_read` с параметрами `chapter_number`, `chapters_read_count`, `book_read_percent`.
7. После всех 15 основных глав появляется `book_completed`.

Если нужно повторно протестировать `chapter_read` в том же браузере, очистите `localStorage` для сайта или удалите ключи:

```text
reader:book-started:v1
reader:read-chapters:v1
reader:book-completed:v1
reader:id:v1
```
